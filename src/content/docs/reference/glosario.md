Este glosario define terminos clave para unificar el lenguaje de la documentacion y evitar ambiguedades.

## Objetivo

- Alinear vocabulario tecnico entre equipos.
- Evitar sinonimos que describen el mismo concepto.

## Convencion general

- Usar **espanol** para explicaciones.
- Mantener **ingles** para nombres tecnicos, APIs, librerias y conceptos ampliamente estandarizados.
- En la primera aparicion de un termino, incluir su equivalente ES/EN cuando aplique.

## Terminos base

| ES | EN | Definicion |
| --- | --- | --- |
| Entorno de desarrollo | Development environment | Configuracion local para ejecutar y probar el proyecto. |
| Integracion continua | Continuous Integration (CI) | Pipeline automatizado que valida cambios. |
| Despliegue continuo | Continuous Deployment (CD) | Publicacion automatica de cambios en entornos definidos. |
| Ajuste final | Final adjustment | Ultima etapa de correcciones antes de cierre. |
| Carrito | Cart | Contenedor de productos seleccionados para compra. |
| Lista de deseos | Wishlist | Lista de productos guardados para futura compra. |
| Tipo de producto | Product type | Estructura base de atributos y variantes de un producto. |
| Encabezado | Header | Seccion superior de la interfaz de una pagina. |
| Pie de pagina | Footer | Seccion inferior de la interfaz de una pagina. |
| Analitica | Analytics | Medicion de eventos, trafico y comportamiento de usuarios. |

## Recomendaciones de uso

- Evitar mezclar dos traducciones para el mismo termino dentro de una misma guia.
- Priorizar el termino definido en este documento al crear nuevo contenido.
- Si un termino no existe, agregarlo aqui antes de adoptarlo en varias guias.
